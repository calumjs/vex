## The 119× speedup

`734298d` ("Optimize embedding pipeline: 53s → 450ms") is the single most load-bearing commit in this period. The commit message records a 119× reduction in cold query latency — achieved by parallelising the CPU path and tightening the BM25 funnel so the neural model only scores a few dozen chunks per query, even on large trees. Without this change, the indexless design would not hold up: a 53-second cold query forces users into a pre-built index. Sub-second cold means query-time embedding is viable in production.

## Hybrid search: `--literal` and path boosts

`84ba552` added `--literal <TERM>`, a repeatable flag that boosts files containing a specific keyword before the neural ranker runs — the escape hatch when the user already knows the vocabulary (`vex "race conditions" --literal lock --literal mutex`). The same commit introduced path-based candidate boosting: matches in `src/` outrank matches in `tests/` or `target/` without needing an explicit filter.

## NPU support, and a caching feature that didn't survive

From the commit messages alone, the rationale appears to be that the cache-invalidation story cost more than it bought given how fast the cold path had become after `734298d`. Worth recording: this was a real merged feature that shipped and was taken back out, not a proposal. The archive preserves both.

## GitHub sync: code and discussions in one search

`fc56053` introduced `vex sync github`, which materializes issues and PRs as Markdown under `~/.local/share/vex/sources/github/owner/repo/` with YAML frontmatter. `42c85cb` then made synced issues appear automatically in every `vex` search — querying "why did we change the auth flow" returns the PR discussion alongside the implementation. Sync is incremental via a watermark in `.meta/sync-state.json`; `--force` resets it.
